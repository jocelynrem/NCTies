DevTools workflow

Mac: Cmd + Option + I PC: Ctrl + Shift + I

ink_selection Select element: click this, then click a part of the page to find its name/class.

devices Toggle device toolbar: preview how the page looks on phone/tablet widths.

Action 1: Find the element name for AI

  1. Open DevTools (shortcut above or Right-click → Inspect)
  2. Use Select element, then click the part you want to change
  3. Find the class or element name in the inspector
  4. Ask AI to edit only that named part
  5. Paste AI's update into your file → refresh

Action 2: Check different screen sizes

  1. Click Toggle device toolbar
  2. Switch between phone/tablet widths
  3. Ask AI to adjust spacing or layout if anything looks off

Accessibility: start with these 4 rules

Think of accessibility as making sure everyone can use your page. If you do these four things, you are off to a strong start.

  1. 1

    Use clear headings: one H1, then H2, then H3 in order.

  2. 2

    Keep text easy to read: use body text around 16px and enough contrast between text and background.

  3. 3

    Make keyboard use possible: users should be able to move with Tab and see the focus outline.

  4. 4

    Use simple motion: avoid busy animations and respect prefers-reduced-motion.

Quick test: press Tab from the top of the page and check that every interactive item gets a visible focus state.
