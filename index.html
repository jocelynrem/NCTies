Page Design Basics (No coding background needed)

Use this as a cheat sheet when you ask AI to build or edit a page. Learn a few key terms and visual patterns so your prompts are clear.

Page structure Design systems Padding vs margin UI elements Accessibility How to ask AI

1) Basic page structure

Think of a page as a simple stack: header, main, and footer. Inside main, you usually have multiple section blocks.

header
Contains h1 (main title) + top nav
main
Parent container for page content
section
Child block inside main
h2
Section title inside section
footer
Bottom area (contact info, links)

Nesting path example: main > section > h2

DevTools shortcut

Open DevTools and click Select element to get the exact element name (or class/id), then paste that name into your AI prompt.

Heading

h1 -> h2 -> h3

Title -> section title -> sub-section title

Page area

header / main / footer

Top, content, bottom structure

Nesting

parent -> child

Example: main contains section, and section contains h2

Targeting hook

class / id

Names AI can target exactly

Vibe coder tip

Use simple location + action prompts:

  • "Change the h1 in the header."
  • "Update the button with class .cta."
  • "Make the second section in main two columns."

If AI misses the target, include the parent + child path (example: main > section > h2).

2) Design systems (the simple version)

A design system is a shortcut to a better-looking page. Instead of styling everything from scratch, you use pre-built layout and UI styles (buttons, cards, spacing) so your page looks polished fast.

Framework

Bootstrap

Best when you want a polished, \"finished app\" look quickly.

Framework

Bulma

Best when you want clean layout with easy-to-read class labels.

Framework

Pico

Best when you want simple, minimal code with great default text styling.

Why this helps with AI coding: AI already understands these design systems. You can ask it to \"build this with Bootstrap,\" or \"swap this page to Bulma/Pico,\" and it can restyle the same content much faster and more consistently.

3) Padding vs Margin (easy rule)

Keep using this simple rule: padding is space inside a box, margin is space outside a box. Using the right word in your AI prompt helps AI change the correct spacing faster.

Margin (outside space)
Padding (inside space)
Content (text, button, image)

Prompt wording you can copy

  • "Add more padding inside this card."
  • "Add more margin-top between these sections."

Quick choice: if content feels cramped, use padding. If sections feel too close together, use margin.

Simple rule: padding is inside, margin is outside. This works the same in Bootstrap, Bulma, and Pico.

5) Quick accessibility checklist

Accessibility means making your page easier for more people to read, use, and navigate.

Treat this as a quick confidence check, not a full audit. If something looks off, ask AI to fix that one item.

  1. Use one h1, then clear h2 section headings.
  2. Keep body text easy to read (about 16px).
  3. Make sure text stands out from its background.
  4. Make links and buttons easy to see and understand.
  5. Press Tab and check that focus is always visible.

Fast check

Press Tab from the top of the page. If any button or link is hard to track, ask AI to improve the focus style for that part.

6) Prompt AI in plain language

Clear intent beats technical jargon. You do not need \"developer words\" to get good results.

A strong beginner prompt has 3 parts: what to change, where to change it, and what to keep the same.

Example prompt

In the resources section card, add more padding inside each card and more margin between sections.
Keep the current colors and typography.
Only edit the card and spacing styles.
Tip: If you are unsure what to edit, open DevTools and use \"Select element\" to find the exact page part name first. Adding that name to your prompt helps AI make a precise change in the right place.
